<objective>
Eliminate hardcoded absolute tool paths (`/usr/bin/zip`, `/usr/bin/unzip`, `/usr/bin/find`, `/usr/bin/who`) from `constantes.sh` by resolving them at load time with `command -v` and a bare-name fallback. Mirror the pattern already used for `DEFAULT_TAR` (and `ssh` in the network layer). Tighten the redundant empty-default redeclarations in the three consumer modules so they never silently collapse the resolved value to empty.

Purpose: Fixes anti-pattern #1 — broken tool resolution on systems where binaries live outside `/usr/bin` (e.g. Homebrew/macOS, custom PATHs), preserving the existing security baseline (`umask 077`, `0600`, config-injection defense).

Output: `constantes.sh` exports resolved tool variables; no literal `/usr/bin/<tool>` strings remain anywhere under `binarios/`.
</objective>

Avoids hardcoding a single absolute path. Operators control `$PATH`; document that PATH must be trusted. |
| T-01-SC | Tampering | shellcheck/package installs (npm dev tooling) | mitigate | No new package installs in this plan; package-lock.json already pins dev deps. If `npx bats` fetches, it uses the pinned `bats@^1.13.0`. |
