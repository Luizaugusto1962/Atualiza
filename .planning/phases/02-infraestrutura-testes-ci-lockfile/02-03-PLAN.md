# 02-03: Lockfile com flock

## Objective

Implementar lockfile com `flock` no entry point `atualiza.sh` para evitar execução concorrente do sistema, seguindo o padrão `exec 200>/var/lock; flock -n 200`.

## Tasks

### Task 1: Implementar lockfile no entry point

<read_first>
- atualiza.sh (entry point — entender fluxo atual)
- binarios/principal.sh (onde o sistema inicializa)
- binarios/constantes.sh (variáveis de configuração)
</read_first>

<action>
- Em `atualiza.sh`, antes de chamar `principal.sh`:
  - Definir `LOCKFILE` como variável no topo (ex: `/tmp/atualiza.lock`)
  - Adicionar `exec 200>"$LOCKFILE"` para abrir fd
  - Adicionar `flock -n 200 || { echo "Outra instância em execução."; exit 1; }`
  - Adicionar trap no EXIT para garantir limpeza (embora flock libere automaticamente)
- Registrar o PID no lockfile para diagnóstico: `echo $$ >&200`
- Adicionar flag `--force` para ignorar lock (útil para manutenção)
</action>

<acceptance_criteria>
- `atualiza.sh` contém lockfile com flock antes de qualquer operação
- Rodar `./atualiza.sh` duas vezes simultaneamente: segunda instância exibe "Outra instância em execução." e sai com exit 1
- `./atualiza.sh --force` ignora o lock e executa normalmente
- Lock é liberado automaticamente quando o processo termina (flake no flock)
</acceptance_criteria>
