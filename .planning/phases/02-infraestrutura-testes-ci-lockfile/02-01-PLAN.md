# 02-01: Testes automatizados com bats

## Objective

Vendorizar bats-core no repositório e escrever testes bats para as funções extraídas no v1.0 em `backup.sh`, `programas.sh` e `arquivos.sh`.

## Tasks

### Task 1: Vendorizar bats-core

<read_first>
- README.md (estrutura do projeto)
- .gitignore (para não ignorar vendor bats)
- package.json (se existir script de test)
</read_first>

<action>
- Criar diretório `.test/bats/`
- Clonar bats-core, bats-support e bats-assert com `--depth 1` para `.test/bats/`
- Remover diretórios `.git` internos
- Adicionar `.test/` ao `.gitignore`? (decidir — recomendado versionar)
- Adicionar script `npm test` rodando `bats test/*.bats` se `package.json` existir
</action>

<acceptance_criteria>
- Diretório `.test/bats/bats-core/bin/` existe com executável `bats`
- `.test/bats/bats-support/load.bash` existe
- `.test/bats/bats-assert/load.bash` existe
- `bats --version` retorna OK via `./.test/bats/bats-core/bin/bats`
</acceptance_criteria>

### Task 2: Testar funções extraídas de backup.sh

<read_first>
- binarios/utils.sh (funções extraídas: _validar_integridade_backup, _executar_backup_comum, etc.)
- binarios/backup.sh (callers das funções)
</read_first>

<action>
- Criar `.test/tests/backup-utils.bats`
- `setup()` deve carregar bats-support, bats-assert e source dos módulos necessários
- Escrever testes para:
  - `_validar_integridade_backup` com arquivo válido e com arquivo corrompido
  - `_executar_backup_comum` com diretório fonte e destino
  - `_diretorio_trabalho` com path válido e inválido
- Usar `run` e `assert_success`/`assert_failure` nos testes
</action>

<acceptance_criteria>
- Arquivo `.test/tests/backup-utils.bats` criado com pelo menos 3 test cases
- `bats .test/tests/backup-utils.bats` retorna 0
- Cada test case testa um comportamento específico (success/failure)
</acceptance_criteria>

### Task 3: Testar funções de programas.sh e arquivos.sh

<read_first>
- binarios/programas.sh (funções com preâmbulo extraído)
- binarios/arquivos.sh (função _listar_logs)
</read_first>

<action>
- Criar `.test/tests/programas-utils.bats` — testar `_listar_logs` com parâmetros `atualizacao` e `limpeza`
- Criar `.test/tests/arquivos-utils.bats` — testar `_validar_arquivo_existe`, `_mudar_diretorio`
- Usar `setup_file` para configurar diretórios temporários
- Usar `teardown_file` para limpar
</action>

<acceptance_criteria>
- `.test/tests/programas-utils.bats` existe com 2+ test cases
- `.test/tests/arquivos-utils.bats` existe com 2+ test cases
- `bats .test/tests/` roda todos os testes e retorna 0
</acceptance_criteria>

## Verification

- `./.test/bats/bats-core/bin/bats .test/tests/` retorna exit 0
- Pelo menos 5 test cases no total
- Testes cobrem success e failure paths
