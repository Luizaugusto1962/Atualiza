# 02-02: Pipeline CI com ShellCheck

## Objective

Configurar GitHub Actions para rodar ShellCheck em todos os módulos .sh em todo push/PR, com `.shellcheckrc` para configuração de regras.

## Tasks

### Task 1: Criar .shellcheckrc

<read_first>
- binarios/*.sh (amostra de 2-3 arquivos para entender o estilo)
- AGENTS.md (convenções do projeto)
</read_first>

<action>
- Criar `.shellcheckrc` na raiz do projeto com:
  - `disable=SC1090,SC1091` (sourcing dinâmico — falso positivo comum em módulos)
  - `source-path=binarios/` (para resolver sourced files)
- Comentar cada disable com o motivo
</action>

<acceptance_criteria>
- `.shellcheckrc` existe na raiz
- `shellcheck --severity=warning binarios/config.sh` passa sem erros
</acceptance_criteria>

### Task 2: Criar GitHub Action

<read_first>
- (nenhum workflow existente — primeiro CI)
</read_first>

<action>
- Criar `.github/workflows/shellcheck.yml`:
  - Trigger: `push` e `pull_request` em `main`
  - Job `shellcheck`:
    - `runs-on: ubuntu-latest`
    - Steps: checkout@v4 → Run ShellCheck
    - Usar `ludeeus/action-shellcheck@master` com `severity: warning`
    - `ignore_paths: ./vendor ./node_modules`
  - Permissions mínimas (`contents: read`)
</action>

<acceptance_criteria>
- `.github/workflows/shellcheck.yml` existe
- Workflow roda em `push` e `pull_request` na branch `main`
- Usa `severity: warning` como gate
- Ignora vendor e node_modules
</acceptance_criteria>

### Task 3: Correções iniciais de ShellCheck

<read_first>
- binarios/*.sh (todos os módulos)
</read_first>

<action>
- Rodar `shellcheck --severity=warning binarios/*.sh`
- Para cada warning:
  - Corrigir se for bug real (quoting, variáveis não utilizadas, etc.)
  - Adicionar `# shellcheck disable=SCxxxx` com comentário justificando se for falso positivo
- Priorizar: errors primeiro, depois warnings
- Focar em: SC2086 (double quote), SC2154 (variável referenciada), SC2034 (variável não usada)
</action>

<acceptance_criteria>
- `shellcheck --severity=warning binarios/*.sh` retorna 0
- Todos os disables têm comentário explicando o motivo
- Nenhum disable global — cada supressão é inline
</acceptance_criteria>

## Verification

- `.github/workflows/shellcheck.yml` existe com configuração válida
- `shellcheck --severity=warning binarios/*.sh` retorna 0
- `.shellcheckrc` presente com configuração do projeto
